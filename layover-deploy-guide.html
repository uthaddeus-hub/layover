Before you start
1
📁 Gather your files
Make sure you have all 8 HTML files downloaded from Claude. They should all be in the same folder on your computer.
📄layover-landing.htmlPublic homepage + waitlistEntry point
📄layover-login.htmlSign in / sign up
📄layover-dashboard.htmlHome dashboard
📄layover-trip-detail.htmlTrip itinerary + chat
📄layover-messages.htmlDirect messages
📄layover-friends.htmlFriends + discover
📄layover-profile.htmlUser profile
📄layover-explore.htmlCity explorer

💡 Create a folder on your Desktop called layover and put all 8 files inside it. This is your project folder.

2
📝 Create an index file
Vercel needs a file called index.html as the entry point — the page that loads when someone visits your URL. Create a new file called index.html in your layover folder and paste this into it:
<!-- index.html -->
<!-- This just redirects visitors to your landing page -->
<meta http-equiv="refresh" content="0; url=layover-landing.html">

This tells any browser that hits your root URL to immediately load layover-landing.html. That's it — you only need these 3 lines.

Deploy
3
🔗 Create a GitHub account (if needed)
Vercel deploys from GitHub. If you don't have a GitHub account, go to github.com and sign up for free — it takes 2 minutes.
→Go to github.com/signup
→Create a free account with your email
→Verify your email address
4
📦 Create a GitHub repository
A repository (repo) is like a folder on GitHub that stores your project files. Once your files are in a repo, Vercel can see them.
1Click the + button (top right on GitHub) → New repository
2Name it layover
3Set it to Public
4Check Add a README file
5Click Create repository
5
⬆️ Upload your files to GitHub
You don't need to use the command line. GitHub has a drag-and-drop file uploader built right in.
1Inside your new repo, click Add file → Upload files
2Drag all 9 files from your layover folder into the upload area (the 8 HTML files + index.html)
3Scroll down and click Commit changes

💡 GitHub will show a loading bar as it uploads. Once complete, you'll see all 9 files listed in your repo. That means it worked.

6
▲ Deploy on Vercel
Now the easy part. Vercel connects to your GitHub repo and publishes your site automatically.
1Go to vercel.com and click Sign Up
2Choose Continue with GitHub — this links your accounts
3Click Add New → Project
4Find your layover repo in the list and click Import
5Leave all settings as-is (no build commands needed for HTML files)
6Click Deploy

Vercel will build and deploy in about 30 seconds. When it's done, you'll see a green checkmark and a live URL that looks like layover-yourname.vercel.app.

Polish
7
🔐 Update Supabase allowed URLs
Supabase only sends password reset emails and auth redirects to URLs you've pre-approved. Once you have your Vercel URL, add it to Supabase.
1Go to supabase.com → your project → Authentication → URL Configuration
2Under Site URL, replace http://localhost with your Vercel URL, e.g. https://layover-yourname.vercel.app
3Under Redirect URLs, add the same URL
4Click Save

⚠️ If you skip this step, password reset emails will link to localhost and won't work for real users.

8
🌐 Add a custom domain (optional)
Want layover.travel instead of layover-abc123.vercel.app? You can buy a domain and connect it for free.
→Buy a domain at namecheap.com or porkbun.com (~$10–12/year)
→In Vercel: Project Settings → Domains → Add domain
→Copy the DNS records Vercel gives you
→Paste them into your domain registrar's DNS settings
→Wait 10–30 min — Vercel handles the SSL certificate automatically
9
🚀 Ship updates instantly
Once you're set up, deploying new changes takes 3 clicks. Every time you push a file to GitHub, Vercel redeploys automatically.
✓Edit a file (e.g. change text on the landing page)
✓Go to your GitHub repo → Upload files → drag the updated file in → Commit changes
✓Vercel detects the change and redeploys in ~30 seconds

💡 Once you get comfortable, learn Git and the Vercel CLI. You'll be able to deploy with a single command from your terminal: vercel --prod

Supabase setup checklist
Make sure these tables exist in your Supabase project before sharing with real users. Go to supabase.com → your project → Table Editor → New Table.
🗄 waitlist Columns: email (text, unique), created_at (timestamptz, default now()) Already built
🗄 trips Columns: user_id (uuid), city (text), depart_date (date), return_date (date), notes (text), visibility (text) Already built
🗄 Row-Level Security Enable RLS on trips table, add policy: auth.uid() = user_id Do before launch
